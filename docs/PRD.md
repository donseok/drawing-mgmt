## 1. 배경 및 목적

### 1.1 배경
- 기존 TeamPlus 시스템 노후화 (클라이언트 설치 의존, ActiveX/COM 기반 추정, Windows 전용 인쇄 의존, 모바일 미지원)
- 매뉴얼 분석 결과 **DWG 뷰어가 별도 ACAD Viewer 호출**, **일괄등록은 별도 EXE**, **Microsoft Print to PDF 의존** 등 사용성·유지보수성 저하 요인 다수
- 사용자가 **두 개의 파일을 직접 실행**해야 뷰어가 동작하는 등 운영 부담 누적
- 사내 표준 브라우저(Chrome/Edge) 환경에서 클라이언트 설치 없이 동작하는 현대적 웹 시스템으로 재구축 필요

### 1.2 목적
- **Pure Web** 기반의 도면관리시스템 구축 (브라우저만으로 DWG 보기·검색·결재 가능)
- 무료 오픈소스 SW 만으로 구성하여 **라이선스 비용 0원** 달성
- 1인이 유지보수 가능한 단순한 아키텍처 설계
- 향후 그룹웨어 결재·DLP·외부 협력업체 포털 확장 가능한 구조 확보

### 1.3 적용 범위
- 동국씨엠 사내 LAN 환경 (온프레미스 단일 서버)
- 사용자 20명 이내, 동시접속 5명 이내, 협력업체 5개 이내
- 기존 TeamPlus 데이터 전수 마이그레이션

## 2. 시스템 개요

### 2.1 시스템 정의
**EDMS(Engineering Document Management System)** — 도면·문서를 데이터베이스화하여 검색·등록·수정·출력·결재·이력관리를 통합 제공하는 웹 시스템.

### 2.2 핵심 가치
1. **설치 없는 웹 뷰어** — 사용자가 별도 클라이언트 설치 없이 DWG 도면을 브라우저에서 즉시 확인
2. **단일 코드베이스 / 단일 서버** — 1인 유지보수 가능한 단순 구조
3. **라이프사이클 보존** — 기존 TeamPlus의 체크아웃·체크인·승인 흐름 유지로 사용자 학습 부담 최소화
4. **권한 기반 보안** — 자료/폴더/기능 권한을 매트릭스로 분리

### 2.3 사용자 군 (Persona)

| 역할 | 인원 | 주요 작업 |
|---|---|---|
| 슈퍼관리자 | 1~2 | 시스템 전체 설정, 권한, 폴더 구조, 자료유형 정의 |
| 관리자 | 2~3 | 사용자·조직·그룹 관리, 자료 일괄등록, 자료 이동 |
| 일반 사용자 (설계자) | 10~15 | 도면 신규등록·수정·승인요청, 검색, 뷰어 |
| 일반 사용자 (열람자) | 5~10 | 검색·뷰어·출력·다운로드 |
| 협력업체 | 5개사 이내 | 로비함을 통한 도면 확인·재확인요청 |

## 3. 사용자 시나리오 (Top 5)

### 3.1 설계자 — 도면 신규 등록
1. 폴더를 선택하고 [신규등록] 클릭
2. 도면번호 자동발번 확인, 자료유형 자동 선택
3. DWG 파일 첨부 → 첫 파일이 마스터파일 ("M" 표시)
4. 보안등급·연결문서 입력 후 [등록] 클릭
5. 시스템이 비동기로 DWG → DXF/PDF 변환 (사용자는 즉시 다음 작업 가능)

### 3.2 설계자 — 도면 개정
1. 검색 → 자료 선택 → [개정] 클릭 (전제: 승인 상태)
2. 자료 상태가 Check-out으로 전환, 다른 사용자 잠금
3. 수정 후 [체크인] → 버전 +0.1
4. [승인요청] → 결재선 지정 → 상신 → 결재자 알림
5. 최종 결재 완료 시 Revision +1, 승인 상태 전환

### 3.3 일반 사용자 — 도면 검색·열람
1. 검색바에 도면번호/키워드 입력
2. 자료유형·폴더·등록일·등급으로 필터
3. 결과 더블클릭 → 웹뷰어로 즉시 표시
4. 확대/축소·회전·측정·문자검색 활용
5. 필요 시 PDF 변환·인쇄·다운로드

### 3.4 관리자 — 일괄등록
1. 관리자 메뉴 → 일괄등록
2. Excel 템플릿 다운로드 → 메타정보 입력
3. DWG 파일과 Excel 템플릿 동시 업로드 (드래그&드롭)
4. 매핑 미리보기 → 검증 통과 항목만 [등록]
5. 변환 작업 큐에 적재, 진행률 실시간 표시

### 3.5 협력업체 — 로비함 도면 확인
1. 협력업체 계정으로 로그인 → 로비함 메뉴만 표시
2. 자사 대상 로비함 목록 확인
3. 도면 미리보기 → 확인의견 입력
4. [확인요청] 또는 [재확인요청] 클릭
5. 만료기간 도래 시 시스템이 자동 폐기

## 5. 비기능 요구사항

### 5.1 성능

| 지표 | 목표 |
|---|---|
| 페이지 로드 | < 2초 (목록·검색) |
| DWG 변환 | < 30초 (10MB 미만 도면 95%) |
| 변환 캐시 응답 | < 500ms |
| 동시 변환 작업 | 3 worker 병렬 |
| 검색 응답 | < 1초 (10만 건 메타) |

### 5.2 가용성·운영

## 6. 도면번호 체계 (신규 설계 제안)

> 기존 체계가 없으므로 새로 정의. 관리자 메뉴에서 변경 가능.

```
[폴더코드]-[유형코드]-[연도]-[일련번호]
예: CGL-MEC-2026-00012
    └ CGL : 폴더(라인)코드
      └ MEC : 자료유형 (MEC=기계, ELE=전기, INS=계장, PRC=공정, GEN=일반)
        └ 2026 : 등록 연도
          └ 00012 : 5자리 일련번호 (연도별 리셋)
```

- 폴더코드: 라인/공정 단위 (예: CGL, SPM, CCL, PCL, COM)
- 일련번호 자릿수와 초기값은 자료유형별 설정 가능
- 자동발번 ON/OFF 토글, OFF 시 수동 입력

## 9. 가정·전제·제약

### 9.1 가정
- DWG 파일은 AutoCAD 2018 이후 포맷 (ODA File Converter가 변환 가능한 범위)
- 사내 폰트는 표준 SHX (romans, txt 등) + 한글 폰트 1~2종
- 단일 서버 (CPU 8코어·RAM 16GB·SSD 500GB·NAS 백업) 가정
- 사내 LAN, HTTPS 인증서 사내 발급 가능

### 9.2 제약
- 라이선스 비용 0원 (모든 SW는 OSS·무상 배포 한정)
- 1인 개발 4주 (휴일 제외 약 20영업일)
- 모바일 미지원
- 외부 인터넷 노출 없음 (협력업체도 사내 VPN 또는 게스트망 가정 — 운영 정책으로 결정 필요)

### 9.3 Out of Scope (Phase 1 제외)
- 그룹웨어 결재 통합
- LDAP/AD SSO
- DLP / 워터마크 / 화면캡처 방지
- OCR (스캔 도면 본문 검색)
- 외부 협력업체 포털 분리
- 도면 Diff 비교 시각화
- BIM(IFC) 지원
