## 1. 디자인 원칙

1. **Drawing-first** — 검색·미리보기·뷰어가 가장 빠르게 도달되어야 한다. 메인 작업 동선은 3클릭 이내.
2. **Dense by default** — 엔지니어 사용자(설계자) 대상. 정보 밀도 우선, 패딩보다 데이터.
3. **Keyboard-first** — 검색·필터·이동은 단축키로 완결. 리스트는 ↑↓·Enter·Space 표준 동작.
4. **Status at a glance** — 자료 상태(NEW/CHECKED_OUT/APPROVED 등)는 색·뱃지·아이콘 3중으로 식별 가능.
5. **Progressive disclosure** — 권한·이력·결재선 같은 보조 정보는 우측 패널·탭·드로어로 격리.
6. **Truthful affordance** — 권한 없는 동작은 비활성/숨김. 회색 처리 시 reason tooltip 필수.
7. **Predictable** — 동일 동작은 어디서든 동일 단축키·동일 위치. 우측 상단 = 글로벌 액션, 우측 하단 = 챗봇.
8. **No surprises** — 변경 동작(체크인·결재·삭제)은 항상 확인 모달. 자동 저장은 명시적 토스트.

## 2. 디자인 시스템

### 2.1 컬러 팔레트 (Tailwind 토큰 기반)

#### Brand
| 토큰 | Light | Dark | 용도 |
|---|---|---|---|
| `brand-50` | #eff6ff | #0b1220 | 배경 강조 |
| `brand-500` | **#2563eb** | **#3b82f6** | Primary CTA |
| `brand-600` | #1d4ed8 | #2563eb | Hover |
| `brand-700` | #1e40af | #1d4ed8 | Active |

> Brand는 도면(blueprint) 톤의 cobalt blue. 채도가 높지 않아 장시간 사용에 부담 적음.

#### Semantic / Status (자료 상태와 1:1 매핑)
| Status | Color | Tailwind | Usage |
|---|---|---|---|
| `NEW` | Slate | `slate-500` | 신규 등록, 미결재 |
| `CHECKED_OUT` | Amber | `amber-500` | 체크아웃(잠금 중) |
| `CHECKED_IN` | Sky | `sky-500` | 체크인 완료 |
| `IN_APPROVAL` | Violet | `violet-500` | 결재 진행 중 |
| `APPROVED` | Emerald | `emerald-500` | 승인됨 |
| `REJECTED` | Rose | `rose-500` | 반려됨 |
| `DELETED` | Stone | `stone-500` | 폐기함 |

#### Neutral (Surface)
- `bg`: `white` / `zinc-950`
- `bg-subtle`: `zinc-50` / `zinc-900`
- `bg-muted`: `zinc-100` / `zinc-800`
- `border`: `zinc-200` / `zinc-800`
- `border-strong`: `zinc-300` / `zinc-700`
- `fg`: `zinc-900` / `zinc-50`
- `fg-muted`: `zinc-600` / `zinc-400`
- `fg-subtle`: `zinc-500` / `zinc-500`

#### 의미 색
- `success`: emerald-500
- `warning`: amber-500
- `danger`: rose-600
- `info`: sky-500

### 2.3 간격·반경·그림자

| Token | Value |
|---|---|
| Space scale | 4 / 8 / 12 / 16 / 20 / 24 / 32 / 48 / 64 |
| Radius `sm` | 4px (입력 필드, 인라인 뱃지) |
| Radius `md` | 6px (버튼, 카드) |
| Radius `lg` | 8px (모달, 패널) |
| Shadow `sm` | 그림자 거의 없음 (border 우선 사용) |
| Shadow `md` | `0 4px 12px rgb(0 0 0 / 0.08)` (드로어, 팝오버) |
| Shadow `lg` | `0 12px 32px rgb(0 0 0 / 0.12)` (모달) |

> 그림자보다 **1px border**를 기본으로 한다. 엔지니어 도구의 dense-and-flat 미감.

### 2.4 아이콘

- **Lucide React** 단일 라이브러리. 16/20/24px 그리드.
- 도면 상태 아이콘은 status color와 함께 페어링 (예: `LockOpen`+amber=CHECKED_OUT)
- DWG/PDF/이미지 파일 아이콘은 mimeType 기반 자동 선택 (`FileType` 컴포넌트)

## 4. 글로벌 레이아웃 (프레임)

### 4.1 그리드

```
┌──────────────────────────────────────────────────────────────────────────┐
│                       Header (56px, sticky, z-40)                         │
├────┬────────────────┬─────────────────────────────────────────┬──────────┤
│Nav │  Sub-Sidebar    │              Main Content              │  Detail  │
│Rail│  (폴더 트리 등)   │                                         │  Panel   │
│56  │   240px          │              fluid                      │  400px   │
│px  │   (선택)         │                                         │  (선택)   │
│    │                 │                                         │          │
│    │                 │                                         │          │
│    │                 │                                         │          │
└────┴────────────────┴─────────────────────────────────────────┴──────────┘
                                                          ╭─[Chat Btn]─╮
                                                          │  64×64 FAB │
                                                          ╰────────────╯
```

- 최소 폭: **1280px**. 1440~1920에 최적화.
- Detail Panel은 자료 상세·결재·로비에서 선택적으로 사용 (resize handle 제공, 320~640px)
- Sub-Sidebar는 화면 컨텍스트에 따라 폴더 트리 / 결재함 메뉴 / 관리 메뉴로 교체

### 4.2 헤더 (Header)

```
┌──────────────────────────────────────────────────────────────────────────┐
│ ▣ 동국씨엠 도면관리   |  [⌘K  도면번호·키워드 검색...]   |  🔔 3   👤 김철수 ▾ │
└──────────────────────────────────────────────────────────────────────────┘
```

- **로고 + 시스템명** (좌, 클릭 → /)
- **글로벌 검색 (⌘K)** — 중앙 가변 폭, 단축키 힌트 노출. 클릭 시 cmdk 팔레트 열림.
- **알림 벨** — 결재 알림·시스템 공지 배지
- **사용자 메뉴** — 이름·조직·로그아웃·다크모드·도움말·프로필

### 4.3 좌측 Nav Rail (1차 네비)

세로 56px, 아이콘 + 툴팁. 활성 항목은 brand-500 좌측 4px 바.

```
┌────┐
│ 🏠 │ Home / Workspace
│ 🔍 │ 자료 검색  (default landing)
│ ✅ │ 결재함
│ 📨 │ 로비함
│ ⭐ │ 즐겨찾기
│ ⚙️ │ 관리       (권한자만)
│────│
│ ?  │ 도움말
└────┘
```

활성 메뉴별 Sub-Sidebar 콘텐츠 자동 전환.

### 4.4 Sub-Sidebar (2차 네비, 컨텍스트별)

#### 자료 검색 화면
- 폴더 트리 (Tree, 가상 스크롤, 1만 노드까지)
- 노드 우클릭: 신규등록·하위폴더 생성·이름변경(권한자)·이동·복사
- 폴더 권한 표시 아이콘 (잠금/공개/제한)

#### 결재함 화면
- 결재함 4종 라디오 리스트: 대기 / 처리완료 / 상신함 / 회수
- 각 항목 옆 카운트 뱃지

#### 관리자 화면
- 메뉴 그룹: `사용자/조직`, `폴더/권한`, `자료유형`, `규칙/공지`, `통합/로그`

### 4.5 Detail Panel (3차, 선택적)

- 자료 상세에서 미리보기·이력·연결문서 동시 보기
- ESC 또는 X로 닫힘. 폭은 사용자 마지막 값 localStorage 저장.

### 4.6 챗봇 위젯 (FAB)

- 우측 하단 64px 원형 버튼 (`MessageSquare` 아이콘)
- 클릭 시 우측 슬라이드 Drawer (480px, 풀높이) 또는 모달 (centered, 720×640)
- 모드 배지(`AI` / `간이`)는 헤더에 표시 (TRD §14 참조)

## 5. 네비게이션 패턴

### 5.1 글로벌 검색 (⌘K Command Palette)

- 세 가지 컨텍스트로 결과 그룹화:
  - **도면**: 도면번호 prefix 일치 → 자료명 trgm
  - **폴더**: 폴더코드·이름
  - **명령**: "내 결재함", "신규등록", "관리자: 사용자", "다크모드 토글"
- 단축키:
  - `⌘K` 열기
  - `↑↓` 이동, `Enter` 실행, `⌘Enter` 새 탭
  - `Tab`으로 그룹 전환
  - 입력 prefix `>` = 명령 모드, `#` = 폴더, `@` = 사용자(공유 등)

### 5.2 브레드크럼 (Breadcrumb)

- 자료 상세·검색에서 폴더 경로 표시
- 각 노드 클릭 가능, 우측 끝에 자료 상태 뱃지

```
폴더: 본사 / 기계 / CGL-1 / 메인라인  > 자료: CGL-MEC-2026-00012  [APPROVED ●]
```

### 5.3 탭

- 자료 상세 5탭은 url query param (`?tab=`)으로 라우팅 → 새 탭으로 공유 가능
- 탭 전환은 `[`/`]` 단축키
- 활성 탭 하단 2px brand 라인

### 5.4 페이지네이션·무한스크롤

- 검색 결과: **무한 스크롤** (50건/페이지). "더 보기" 버튼 + 자동 로드 둘 다 지원
- 결재함: **페이지네이션** (30건/페이지). 결재함은 처리 단위로 묶여 있어 페이지가 더 자연스러움.

- `get_help` → `MarkdownCard` (RAG 컨텍스트 출처 footer 포함)

### 8.4 인터랙션
- 메시지 hover 시 `복사`, `다시 생성`(RAG only), `링크 공유`(세션 ID 포함)
- 빠른 답변 칩(quick replies)은 Rule 모드에서 항상 표시, RAG 모드는 첫 메시지 후 1회만
- 코드 블록은 Mono + 복사 버튼
- 출처(citations): RAG 응답 하단에 `[1] manual:checkout` 링크

|---|---|

(Vim 스타일 더블 키 `g+x`는 1.5초 내 입력 시 동작)

### 9.2 토스트 정책

| 종류 | 위치 | 자동 소멸 | 색 |
|---|---|---|---|
| Info | 우상단 | 4s | sky |
| Success | 우상단 | 4s | emerald |
| Warning | 우상단 | 6s | amber |
| Error | 우상단 | persist (X) | rose |

토스트 메시지는 단문 (≤ 60자) + 옵션 액션 (1개 이내).

### 9.3 확인 모달

- 위험 동작(삭제, 폐기, 결재 반려, 권한 변경)은 항상 모달
- "삭제" 같은 단어를 입력하게 하지 않음 (마찰 과다). 대신 **5초 동안 Undo 토스트** 제공 (가능한 경우)
- 모달 primary 버튼 = 위험 동작 전용 색 (`destructive`), default focus는 cancel

### 9.4 자동 저장
- 신규등록·관리자 폼은 5초 idle 시 draft 자동 저장 (localStorage), 토스트로 알림
- 새로고침 시 복원 prompt

## 10. 빈 상태 · 로딩 · 에러

### 10.1 EmptyState 패턴
```
┌──────────────────────────────────────┐
│              [Icon, 48px]             │
│                                       │
│         결과가 없습니다                │
│   필터를 줄이거나 다른 키워드를 사용해│
│   보세요.                             │
│                                       │
│         [필터 초기화]                 │
└──────────────────────────────────────┘
```
- 일러스트 대신 Lucide 아이콘 + 무채색 (engineering tone)
- 컨텍스트별 메시지: 검색결과 없음 / 폴더 비어있음 / 결재 없음 / 권한 없음

### 10.2 로딩
- **Skeleton** 우선 (테이블·카드 형태 보존). spinner는 보조
- 글로벌 라우트 전환은 NProgress 상단 2px brand 라인
- 변환 진행 중 자료는 썸네일 자리에 회전 아이콘 + "변환 중" + 큐 위치(`#3 / 7`)

### 10.3 에러
- 4xx: 인라인 (폼 필드별)
- 5xx: 페이지 단위 ErrorBoundary, "다시 시도" + "관리자에게 문의" + 에러 ID
- 권한 없음: 403 페이지 + 권한 요청 가이드

## 11. 접근성 (A11y)

- WCAG 2.1 AA 준수 목표 (Phase 1: 키보드·콘트라스트·라벨)
- 모든 인터랙티브 요소 `:focus-visible` 2px brand outline
- color는 의미 전달의 보조이며, 아이콘·텍스트와 함께 사용 (색맹 대응)
- aria-label, aria-live(toast), aria-expanded(트리/탭) 일관 적용
- shadcn/ui는 Radix 기반으로 기본 a11y 제공 — 커스텀 컴포넌트만 검수 대상
- 단, 챗봇 모드 배지는 색만으로 구분하지 않음 (`AI`/`간이` 텍스트 병기)

## 12. 다크 모드

- **시스템 prefers-color-scheme + 사용자 토글** (3상: light / dark / system)
- 토큰 기반이라 컴포넌트 별도 구현 불필요
- 도면 미리보기·뷰어는 별도 토글 (도면은 흰 배경이 가독성 ↑) — 다크 모드라도 viewer canvas는 white 유지
- status color는 다크에서도 동일 hue, lightness ↑
